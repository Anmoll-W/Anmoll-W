| Title | Covers 
|---|---|
| [Hermes, Wave 2: The Tests Passed. The Code Was Broken Anyway.](https://github.com/Anmoll-W/blog/blob/main/posts/hermes-wave-2-tests-that-lie.md) | Wave 2 of the Hermes series: consolidating a sprawl of scattered Mac automation into a handful of hardened, always-on jobs on a rented server — and the verification discipline that made it trustworthy. A two-reviewer adversarial gate caught six defects that a green test suite completely missed: a deploy tool that could not deploy its own code, a backup that would have wiped the target on failure, an idempotency ledger that could split and re-fire reminders as spam, a consolidated job that sent the morning digest twice, two tests that passed against deliberately-broken code, and a safety-gate check whose test was deliberately written to pass against the bug it documented. The lesson: a passing test is a claim, and claims get checked by trying to break them. Built and verified by 574 passing checks across nineteen test suites; going live after the current soak. |
| [Hermes, Wave 1: Giving My Vault an Always-On Body](https://github.com/Anmoll-W/blog/blob/main/posts/hermes-the-foundation.md) | The opening post of a new series on Hermes — an always-on AI body for my Obsidian vault, running on a small rented server. Wave 1 is the foundation: a deny-by-default synced slice so private notes never leave the Mac, a single-writer rule that prevents sync conflicts, a fail-loud job harness so nothing dies silently, and dead-man's-switch watchdogs that alert by going quiet. Plus the first two features you can feel — a morning digest texted to Telegram and a reminder loop you text the bot. Every one of the fourteen micro-waves was reviewed by two adversarial AI personas, under a validate-before-fix rule that caught a reminder parser crashing on impossible dates. |
| [How I Taught My Vault to Read YouTube](https://github.com/Anmoll-W/blog/blob/main/posts/youtube-to-vault-pipeline.md) | A yt-dlp and Whisper pipeline that turns any YouTube URL into a vault note. Captions-first, Whisper fallback for audio-only transcription, Claude summarises the transcript. Auto-invoke wired to all six vault personas — any agent can trigger it without manual instruction. Three P0 bugs caught by Vera before ship: VTT metadata leaking into transcripts, stale output file poisoning the next run, and a one-character language selector mismatch that silently bypassed captions on English videos.
| [AI Credits Are Infrastructure. Start Treating Them That Way.](https://github.com/Anmoll-W/blog/blob/main/posts/ai-credits-are-infrastructure.md) | A billing change made AI usage suddenly visible — and the audit that followed showed three cost drivers that PMs already know how to fix. Model routing, session length compaction, and lean automated pipelines. The PM framing: infrastructure cost thinking is an existing skill. The only new thing is actually looking.
| [Upgrading My AI Agent Roster, and the Baseline I Forgot to Save](https://github.com/Anmoll-W/blog/blob/main/posts/personas-180-of-180-and-no-baseline.md) | A depth upgrade on top of the six-persona vault roster: a dissent protocol that requires every agent to steelman an alternative, a fourteen-pattern Anmoll-watchlist, a twelve-phrase banned-list that triggers a gate failure, per-persona refusal catalogs (seven to eight triggers each), per-persona failure-mode lists (ten to fourteen named modes each), and a six-axis structural eval rubric with a department-head bar at twenty-four out of thirty. The eval scored every persona thirty out of thirty — one hundred eighty out of one hundred eighty — and that ceiling is exactly what exposed the problem. The rubric checks file-on-disk properties, not live behavior, and because the vault was never git-snapshotted before the upgrade, there is no pre-upgrade version of any persona to compare against. Ships a forty-seven-line `persona-snapshot.sh` that runs weekly with eight-week retention so the next upgrade has a real before-and-after.
